# 段寄存器

保护模式下一共有六个段寄存器，实模式只有四个

实模式：

CS (Code Segment)：代码段寄存器；
  DS (Data Segment)：数据段寄存器；
  SS (Stack Segment)：堆栈段寄存器；
  ES (Extra Segment)：附加段寄存器；

保护模式：

32 位下没有 GS，GS 是 64 位下用的

段名称后面的数字是段选择子

段是有权限的

LDT： local Descriptor table

GDT： Global Descriptor table

DS == 23 == 0010 0011

RPL==11

TI == 0 // 查 GDT

Index == 0000 0000 0010 0 == 0x04 // 查 GDT 第四项 00cffb00 0000ffff

Segment Limit: f ffff

Base Address: 0000 0000 基址

type: b

s:1 系统段为 0 普通段为 1

DPL:3

P:1 段有效标识，=1 段有效

AVL: 0

D/B:1

G:1

type:

快速定位 gdtr 表中位置 1B (段选择子) & fff8 = x

dq $(gdtr) + x == 00cf9300 0000ffff

CS:1B

base: 00000000

limit: fffff

type: 3

s:1

DPL: 00

P:1

AVL: 0

D/B:1

G:1

# 实验：

[00277000] 逻辑地址

逻辑地址会和 base 相加

如果 base 不是 0

会变成 [00277000 + base] 导致值变了

但是最后会还原 ds 的值，只是中断在那行的时候值会变

gdt hook 改 base

# DB_G

64 位中没有 limit 和 base

1 页 = 4096 字节

G= 1 时，fffff 的单位是页 ，范围 (0xfffff+1) * 0x1000 == 0x100000000 - 1 = 0xffff ffff

G=0 按照字节为单位

# D/B

D/B 描述 CS 时：

D/B = 1 寻址空间为 32 位

D/B = 0 16 位

更变 cs， jmp 0x4b:402344

此时 push 的时候压栈只压 2 字节

D/B 描述数据段时 数据段 + 堆栈段

D/B = 0 16 位，此时 push 的时候程序会直接崩溃，因为系统可访问的最低地址是 0x10000。此时 limit 描述的范围不能访问

向上扩展： 没有描述的区域可以访问

向下扩展：描述的区域可以访问

保护模式分为

段页模式

纯段模式

一致代码段：R3 直接调用 R0
